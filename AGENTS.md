# Agent Guidelines for BeroepsbelgWeb

This document provides instructions for AI agents and developers working on this Next.js codebase.

## 1. Environment & Commands

### Build & Verification
- **Dev Server:** `npm run dev`
- **Linting:** `npm run lint`
- **Type Checking:** `npm run typecheck` (Runs `tsc --noEmit`)
- **Build:** `npm run build`

### Testing
- **Status:** There are currently **NO** automated tests configured (no Jest, Vitest, Playwright, etc.).
- **Verification:** Agents must rely on:
  1. `npm run typecheck` to ensure type safety.
  2. `npm run lint` to catch stylistic/potential errors.
  3. `npm run build` to verify the application builds successfully.
  4. Manual verification if the agent has browser access capabilities.
- **Do NOT** attempt to run `npm test` or similar commands as they do not exist.

## 2. Code Style & Conventions

### Tech Stack
- **Framework:** Next.js 13+ (App Router).
- **Language:** TypeScript (Strict mode).
- **Styling:** Tailwind CSS + Radix UI (shadcn/ui) + Inline Styles (for specific branding).
- **State Management:** React Context (e.g., `auth-context`, `cart-context`) + Supabase.
- **Internationalization:** `next-intl` (Routes are prefixed with `/[locale]/`).

### Formatting & Naming
- **Indentation:** 2 spaces.
- **Semicolons:** Always use semicolons.
- **File Naming:** kebab-case (e.g., `tour-card.tsx`, `main-nav.tsx`).
- **Component Naming:** PascalCase (e.g., `TourCard`, `MainNav`).
- **Directories:** kebab-case, except for route groups (e.g., `(auth)`) or dynamic routes (e.g., `[locale]`).

### Component Structure
- **Directives:** Use `'use client';` at the top of components requiring client-side interactivity (hooks, event listeners).
- **Props:** Define props with interfaces, exported if reused.
  ```typescript
  interface MyComponentProps {
    title: string;
    isActive?: boolean;
  }
  export function MyComponent({ title, isActive }: MyComponentProps) { ... }
  ```
- **Imports:** Group imports logically:
  1. React / Next.js
  2. Third-party libraries (`next-intl`, `lucide-react`)
  3. Internal Components (`@/components/...`)
  4. Internal Lib/Hooks (`@/lib/...`)

### Styling Patterns
- **Tailwind:** Use utility classes for layout, spacing, and standard typography.
- **Inline Styles:** The project heavily uses inline styles for dynamic values, transitions, and specific branding colors.
  - **Variables:** Use CSS variables like `var(--green-accent)`, `var(--primary-base)`, `var(--text-primary)`.
  - **Example:**
    ```tsx
    <div
      className="transition-all duration-300"
      style={{
        backgroundColor: 'var(--green-accent)',
        boxShadow: 'var(--shadow-small)'
      }}
    >
    ```
- **Radix UI:** Use the accessible primitives provided in `components/ui/`.

### Data Fetching & Supabase
- Use the typed Supabase client.
- Prefer server components for initial data fetching where possible.
- Use `useAuth()` for client-side user session access.

### Internationalization (i18n)
- Wrap strings in `useTranslations()` hook.
- **Example:**
  ```tsx
  const t = useTranslations('common');
  return <span>{t('moreInfo')}</span>;
  ```
- Ensure all visible text is internationalized.

## 3. Workflow for Agents

1.  **Analyze:** Read related files to understand the component hierarchy and data flow.
2.  **Plan:** Propose changes ensuring compatibility with the existing strict TypeScript config.
3.  **Implement:**
    -   Use the existing UI components (`@/components/ui/`) instead of creating new ones if possible.
    -   Respect the inline style system for branding consistency.
4.  **Verify:**
    -   **Always** run `npm run typecheck` after changes.
    -   **Always** run `npm run lint`.
    -   If a build error is suspected, run `npm run build`.
5.  **Refactor:** Ensure no unused imports or variables remain.

## 4. Specific Rules & caveats
-   **Placeholders:** Use `getTourPlaceholder` from `@/lib/utils/placeholder-images` for missing images.
-   **Dates:** Use `date-fns` for date manipulation.
-   **Forms:** Use `react-hook-form` with `zod` for validation.
